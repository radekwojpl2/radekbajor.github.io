Github Pages + Hugo Statyczna Strona W 30 Minut

May 4, 2020

6 mins read

W tym poście postaram sie przedstawić instrukcje jak stworzyć blog z użyciem Hugo i GitHub Pages.
Hugo jest to generator statycznych stron internetowych, użycie takiego narzędzia pozwala na szybie dodawanie i edytowanie treści na stronie.
W połączeniu z GitHub Pages możliwe jest stworzenie strony internetowej w przeciągu 30 minut z darmowym hostingiem, dodatkowym plusem takie strony jest domyślny protokół HTTPS działający z GitHub Pages.

Przydatne Linki

Jak zainstalować Gita.
GitHub.

Wymagania

  • Konto na GitHub
  • Zainstalowany Git
  • Dowolny system operacyjny (w moim przypadku Ubuntu 18.04),
  • Zainstalowany VS Code lub inny edytor.
  • Hugo v0.69.2

Instalacja Hugo

Instalacja Hugo odbywa sie poprzez uruchomienie następujących poleceń w terminalu

    $ sudo apt-get update
    $ sudo apt-get install hugo

Do sprawdzenia instalacji można użyć polecenia

    $ radek@radek-desktop:~$ hugo version
    Hugo Static Site Generator v0.69.2 linux/amd64 BuildDate: 2020-04-24T12:36:21Z

Konfiguracja GitHub Pages

Proces konfigurowania GitHub Pages jest bardzo prosty i znajduje ise po tym linkiem

GitHubPages

Tworzenie bloga

Hugo dostarcza gotowe do użycia CLI. Aby stworzyć nowy projekt w Hugo należy w terminalu użyć polecenia

   $ hugo new site <NazwaProjektu>

Następnie należy przejść do folderu zawierającego stworzony projekt i uruchomić edytor z tego poziomu

    $ cd <NazwaProjektu>
    $ code .

Następnie na głównej stronie Hugo Themes możliwie jest wybranie gotowego szablonu strony.

WAŻNE!! Szablony mogą sie od siebie różnić, należ przeczytać dokumentacje jak poprawnie użyć wybranego szablony.

Na potrzeby tej strony zdecydowałem sie na następujący szablon Ramium. Aby dodać szablon do projektu należy uruchomić

    $ git init
    $ git submodule add https://github.com/rafed/ramium.git themes/ramium

Konfiguracja dodanego szablonu odbywa sie poprzez modyfikacje pliku config.toml w głównym folderze projektu. Więcej informacji znajduje sie na stronie szablonu.

baseURL = "http://example.org/"
#CanonifyURLs=true

languageCode = "en-us"
theme = "ramium"

title = "<Nazwa Strony>"
disqusShortname = "disqus-code"
googleAnalytics = "analytics-code"
summaryLength = 40
pluralizeListTitles = false
enableemoji = false

[params]
    description = "A description for the meta tag of the site"
    googleSearch = "google-search-code"
    showDate = true # make false if dont want to show date
    math = false # best to enable this in the front matter of a page

    tagsInHome = 40
    sectionsInHome = 5
    paginatePostsPerPage = 5
    paginateTagsPerPage = 6

[taxonomies]
    tag = "tags"

[markup.goldmark.renderer]
    unsafe = true

[menu]
    [[menu.main]]
        name = "Home"
        url = "/"
        weight = 1
    
    [[menu.main]]
        name = "Author"
        url = "/author/"
        weight = 3

Po skonfigurowaniu szablonu należy dodać pierwszy post oraz sprawdzić czy strona działa lokalnie. Serwer uruchamia sie lokalnie na domyślnym adresie Serwer lokalny

    $ hugo new content/First-Post/first.md
    $ hugo server -D

W lokalizacji ./content/First-Post powinien pojawić sie plik o następującej strukturze.

    ---
    title: "Test"
    date: 2020-05-04T16:10:17+02:00
    draft: true # flagę draft należy zmienić na false przed zbudowaniem strony
    ---

    #Tutaj należy umieścić treść posta, można umieszczać tutaj zwykłego HTML lub znaczniki  Markdown

Link do Markdown

Ostatnim krokiem jest zbudowanie nasze strony oraz wrzucenie jej do repozytorium stworzonego wcześniej Konfiguracja repozytorium. Aby to zrobić należy uruchomić następując polecenia w terminalu na poziomie projektu.

    $ hugo -t ramium # po -t powinna pojawić sie nazwa używanego szablonu 
    $ git submodule add <LinkDoRepozytorium> public
    $ git add --all
    $ git commit -m "<Treść commita>"
    $ git push

Strona powinna znajdować sie po adresem https://<github_username>.github.io/. Należy pamiętać że nie jest to w 100% działający blog, należy dodać do niego obsługę wszystkich stron, skonfigurować wszystkie media społecznościowe i upewnić sie że wszystko działą. Przykładowa budowa strony znajduje sie w folderze themes/ramium/exampleSite.
